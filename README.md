# HSH-64

64-bit 语义哈希编码扩展方案，与 HSH-32 兼容设计但提供更大的语义空间。

## 编码结构

```text
[feat: 4 bit] + [sim: 52 bit] + [abs: 8 bit] = 64 bit
```

| 字段 | 位数 | 说明 |
|------|------|------|
| feat | 4 bit | 词性/常用词标识，最多 16 类 |
| sim  | 52 bit | 语义相似码，2^52 ≈ 4.5 × 10^15 种状态 |
| abs  | 8 bit  | 簇内完美哈希/位置指纹 |

与 HSH-32（feat4 + sim20 + abs8）相比，HSH-64 将 sim 码从 20 bit 扩展到 52 bit，在保持 64-bit 硬件友好（单个 `u64` 存储、单次 `popcnt` 比较）的同时，大幅提升语义区分能力。

## 适用场景

- 词表规模较大（10 万～100 万词），20-bit sim 碰撞严重
- 对召回率要求高于极致压缩比的场景
- 需要与 HSH-32 并行使用、逐步迁移的场景

## 快速开始

```bash
cd hsh64
cargo test
```

## 设计权衡

| 方案 | 存储 | 比较 | 语义空间 | 推荐场景 |
|------|------|------|----------|----------|
| HSH-32 | 4 B/词 | 一条 popcnt | 2^20 | 小词表、极致压缩 |
| **HSH-64** | 8 B/词 | 一条 popcnt | 2^52 | 大词表、高召回 |
| 128-bit+ | 16 B+/词 | 多条 popcnt | 2^128 | 优势被 PQ/HNSW 稀释 |

## 状态

当前为初始版本，仅包含核心 `HSHCode64` 类型。encoder、MIH 索引、训练脚本等模块后续按需扩展。
